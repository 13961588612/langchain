# 03 协议规范

IntentGate 内部采用统一协议，使 Channel Adapter 与 Agent Backend 互不感知对方存在。

---

## 1. 类型总览

```
入站（客户端 → Gateway → Agent）
  StandardMessage      用户文本/附件
  CardActionEvent      卡片按钮/表单交互

出站（Agent → Gateway → 客户端）
  AgentReply           统一回复信封
    ├─ kind=text       纯文本
    ├─ kind=stream     流式片段
    ├─ kind=card       含 CardIntent
    └─ kind=markdown   Markdown

卡片专用
  CardIntent           结构化卡片意图（Agent 产出）
  CardAction           按钮定义（可选覆盖模板）
```

---

## 2. ChannelType

```python
class ChannelType(StrEnum):
    WECOM = "wecom"
    FEISHU = "feishu"
    DINGTALK = "dingtalk"
    WEB = "web"
    API = "api"
```

---

## 3. StandardMessage

用户消息，经 Gateway 归一化后送 Agent **慢路径**。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `session_id` | string | 是 | 会话唯一键 |
| `channel` | ChannelType | 是 | 来源通道 |
| `user_id` | string | 是 | 用户标识 |
| `text` | string | 否 | 文本内容 |
| `metadata` | object | 否 | 通道扩展字段 |

**metadata 常用键（企微）**：

| 键 | 说明 |
|----|------|
| `req_id` | 回调 req_id，回复时透传 |
| `chatid` | 会话 ID |
| `chat_type` | 1 单聊 / 2 群聊 |
| `bot_id` | 机器人 ID |
| `msgid` | 消息 ID |

**示例**：

```json
{
  "session_id": "wecom:default:bot1_gchat99_2",
  "channel": "wecom",
  "user_id": "zhangsan",
  "text": "查询订单 20260520001",
  "metadata": {
    "req_id": "req_abc123",
    "chatid": "gchat99",
    "chat_type": 2,
    "bot_id": "bot1"
  }
}
```

---

## 4. CardActionEvent

卡片交互事件。Gateway **优先快路径**；仅复杂场景送 Agent。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `session_id` | string | 是 | 会话键 |
| `channel` | ChannelType | 是 | 来源通道 |
| `task_id` | string | 是 | 卡片 task_id |
| `action_key` | string | 是 | 按钮 key |
| `user_id` | string | 否 | 操作用户 |
| `payload` | object | 否 | 扩展数据 |
| `metadata` | object | 否 | req_id 等 |

**示例**：

```json
{
  "session_id": "wecom:default:bot1_gchat99_2",
  "channel": "wecom",
  "task_id": "crm:approve:12345:v1",
  "action_key": "approve",
  "user_id": "lisi",
  "metadata": {
    "req_id": "req_event_456"
  }
}
```

---

## 5. CardIntent

Agent 产出的卡片意图，**不含**客户端原生 JSON。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `template` | string | 是 | YAML 模板名（如 `crm_approve`） |
| `slots` | object | 否 | 槽位键值 |
| `actions` | CardAction[] | 否 | 覆盖模板按钮 |
| `task_id` | string | 否 | 覆盖模板 task_id 表达式 |
| `feedback_id` | string | 否 | 用户反馈 ID |

**CardAction**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | string | 按钮 key，对应 action_key |
| `label` | string | 显示文本 |
| `style` | int | 样式（企微：1 主按钮 / 2 次按钮） |

**示例**：

```json
{
  "template": "crm_approve",
  "slots": {
    "orderNo": "20260520001",
    "orderId": "12345",
    "amount": "9800",
    "shopName": "华东店",
    "applicant": "张三"
  },
  "task_id": "crm:approve:12345:v1"
}
```

---

## 6. AgentReply

Agent 返回的统一回复单元，Gateway 按 `kind` 路由到 Channel Adapter。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `session_id` | string | 是 | 目标会话 |
| `kind` | ReplyKind | 是 | text / stream / card / markdown |
| `text` | string | kind=text | 文本内容 |
| `stream` | StreamChunk | kind=stream | 流式块 |
| `card` | CardIntent | kind=card | 卡片意图 |
| `metadata` | object | 否 | 扩展 |

**StreamChunk**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 流 ID，同 ID 多次推送 |
| `content` | string | 增量内容 |
| `finish` | bool | true 表示结束 |

**流式示例序列**：

```json
{"session_id":"...","kind":"stream","stream":{"id":"s1","content":"正在查询","finish":false}}
{"session_id":"...","kind":"stream","stream":{"id":"s1","content":"订单信息…","finish":true}}
{"session_id":"...","kind":"card","card":{"template":"query_summary","slots":{...}}}
```

## 7. Gateway HTTP API

Base URL：`http://{host}:8090`

### 7.1 健康检查

```
GET /health
```

响应：

```json
{
  "status": "ok",
  "service": "intentgate",
  "agent_backend": "mock"
}
```

### 7.2 协议 Schema

```
GET /api/v1/protocol
```

返回各模型的 JSON Schema 及 Agent 端点约定。

### 7.3 入站消息（调试 / 内部）

```
POST /api/v1/messages
Content-Type: application/json
```

请求体：`StandardMessage`  
响应：`AgentReply[]`（当前同步聚合 SSE；生产由 Adapter 内部流式消费）

### 7.4 入站卡片动作（慢路径）

```
POST /api/v1/actions
Content-Type: application/json
```

请求体：`CardActionEvent`  
响应：`AgentReply[]`

### 7.5 卡片渲染（调试）

```
POST /api/v1/cards/render
Content-Type: application/json
```

请求体：

```json
{
  "channel": "wecom",
  "intent": {
    "template": "crm_approve",
    "slots": {
      "orderNo": "20260520001",
      "orderId": "12345",
      "amount": "9800",
      "shopName": "华东店",
      "applicant": "张三"
    }
  }
}
```

响应：企微原生 `template_card` body（可直接用于 `aibot_respond_msg`）。

## 8. Agent Backend API（约定）

Agent 服务（AgentScope Bridge 等）应实现以下端点，供 Gateway 调用。

### 8.1 对话

```
POST /v1/sessions/{session_id}/messages
Content-Type: application/json
Accept: text/event-stream
```

请求体：`StandardMessage`

响应：SSE 流，每行：

```
data: {"session_id":"...","kind":"stream","stream":{...}}

data: {"session_id":"...","kind":"card","card":{...}}
```

### 8.2 复杂卡片动作

```
POST /v1/sessions/{session_id}/actions
Content-Type: application/json
Accept: text/event-stream
```

请求体：`CardActionEvent`  
响应：同上 SSE `AgentReply` 流。

### 8.3 Agent 侧卡片工具约定

Agent 工具返回 JSON block，`type` 固定为 `card_intent`：

```json
{
  "type": "card_intent",
  "template": "crm_approve",
  "slots": { "orderNo": "20260520001", "orderId": "12345" },
  "task_id": "crm:approve:12345:v1"
}
```

Bridge 将其转换为 `AgentReply(kind=card)`。

## 9. 错误处理

| 场景 | HTTP | Gateway 行为 |
|------|------|--------------|
| 模板不存在 | 404 | 回退文本「卡片渲染失败」 |
| 无 Renderer | 400 | 记录错误，回文本 |
| Agent 超时 | 504 | stream 提示「服务繁忙」 |
| Agent 不可用 | 502 | 降级 mock 或固定话术 |
| update 超时 | — | 告警 + 尝试 proactive 新卡 |

---

## 10. 版本与兼容

- 协议版本通过 `GET /api/v1/protocol` 的 `version` 字段暴露（待 P2 添加）
- 新增 `ReplyKind` 或 `ChannelType` 为向后兼容扩展
- 禁止 Agent 直接输出客户端原生卡片 JSON
