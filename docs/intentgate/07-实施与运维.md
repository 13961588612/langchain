## 3. 本地开发

### 3.1 安装

```bash
cd practice/intentgate
python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux / macOS
source .venv/bin/activate

pip install -e .
```

### 3.2 配置

```bash
cp .env.example .env
```

| 变量 | 说明 | 默认 |
|------|------|------|
| `APP_ENV` | 环境 | development |
| `APP_PORT` | 端口 | 8090 |
| `AGENT_BACKEND` | mock / agentscope / aethermind | mock |
| `AGENT_BACKEND_URL` | Agent 服务地址 | http://127.0.0.1:8100 |
| `AETHERMIND_AGENT_ID` | AetherMind Agent UUID | default |
| `WECOM_BOT_ID` | 企微 Bot ID | — |
| `WECOM_BOT_SECRET` | 企微 Secret | — |
| `WECOM_WS_URL` | WS 地址 | wss://openws.work.weixin.qq.com |
| `SESSION_STORE` | memory / redis | memory |
| `REDIS_URL` | Redis 连接 | redis://127.0.0.1:6379/0 |
| `CARDS_DIR` | 模板目录 | ./cards |

### 3.3 启动

```bash
uvicorn intentgate.main:app --reload --host 0.0.0.0 --port 8090
```

### 3.4 验证

```bash
# 健康检查
curl http://127.0.0.1:8090/health

# 协议 Schema
curl http://127.0.0.1:8090/api/v1/protocol

# 卡片渲染
curl -X POST http://127.0.0.1:8090/api/v1/cards/render \
  -H "Content-Type: application/json" \
  -d '{"channel":"wecom","intent":{"template":"crm_approve","slots":{"orderNo":"20260520001","orderId":"12345","amount":"9800","shopName":"华东店","applicant":"张三"}}}'

# Mock 对话
curl -X POST http://127.0.0.1:8090/api/v1/messages \
  -H "Content-Type: application/json" \
  -d '{"session_id":"test:default:s1","channel":"api","user_id":"u1","text":"hello"}'
```

### 3.5 测试

```bash
pytest tests/ -v
```

## 5. 部署

### 5.1 Docker（示例 Dockerfile 待 P1 添加）

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY pyproject.toml .
COPY intentgate/ intentgate/
COPY cards/ cards/
RUN pip install --no-cache-dir .
ENV CARDS_DIR=/app/cards
EXPOSE 8090
CMD ["uvicorn", "intentgate.main:app", "--host", "0.0.0.0", "--port", "8090"]
```

### 5.2 生产约束

- **每个企微 Bot 一个 Gateway 实例**（单 WS 连接）
- Agent 服务可水平扩展
- Redis 高可用（Sentinel 或云 Redis）
- 密钥：`WECOM_BOT_SECRET` 走环境变量或密钥管理，不入库

### 5.3 进程模型（P1 后）

```
进程 1: intentgate (FastAPI + WeComAdapter 内嵌或 sidecar)
进程 2: agentscope-bridge (8100)
进程 3: redis
```

## 6. 监控与告警

| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| `wecom_ws_connected` | WS 连接状态 | 断开 > 1min |
| `card_update_latency_ms` | 点击到 update | p99 > 4000ms |
| `card_update_timeout_total` | 超 5s 失败 | > 0 |
| `agent_sse_ttfb_ms` | Agent 首字节 | p95 > 3000ms |
| `outbound_rate_limited_total` | 限流次数 | 突增 |
| `fast_path_ratio` | 快路径占比 | < 50% 可优化 |

日志字段建议：`session_id`、`task_id`、`req_id`、`action_key`、`channel`。

## 8. 常见问题

**Q：卡片点击无反应？**  
检查 task_id 是否与发卡一致；是否在 5s 内 update；WS 是否在线。

**Q：流式后卡片发不出？**  
确认 stream 已 `finish=true` 且为单独一条 `aibot_respond_msg`。

**Q：主动推送失败？**  
用户须先在同会话给 Bot 发过至少一条消息。

**Q：Agent 切换怎么测？**  
改 `AGENT_BACKEND` 环境变量，重启 Gateway，接口不变。

**Q：如何新增业务卡片？**  
在 `cards/` 加 YAML → Agent 工具传 template 名 → 无需改 Renderer（同 card_type）。
