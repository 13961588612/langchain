# 01 项目规划

## 1. 背景与问题

企业 IM（企业微信、飞书、钉钉）已成为内部协作主入口。业务系统（CRM、工单、告警）需要在会话内提供**结构化交互**——审批按钮、查询摘要、告警确认——而非纯文本对话。

常见失败模式：

| 问题 | 后果 |
|------|------|
| LLM 直接输出客户端卡片 JSON | 结构错误、难维护、各端不统一 |
| 每次按钮点击都调用完整 Agent | 超时（企微 5 秒更新限制）、成本高、体验差 |
| Gateway 与 Agent 同进程 | 心跳断连、LLM 阻塞通道 |
| 每个 IM 各写一套卡片逻辑 | 重复劳动、状态不一致 |

IntentGate 解决：**在客户端与智能体之间插入统一的卡片交互层**。

---

## 2. 项目定位

**IntentGate** = **Intent**（结构化卡片意图）+ **Gate**（网关）

| 属性 | 说明 |
|------|------|
| 类型 | 独立微服务（Python / FastAPI） |
| 职责 | 多通道消息归一化、卡片渲染、会话状态、Agent 路由 |
| 不负责 | LLM 推理、业务 CRUD、用户权限体系（由 Agent 后端与业务系统承担） |
| 代码位置 | `practice/intentgate/` |

与智能体平台的关系：

```
IM 客户端 ──► IntentGate ──► AgentScope（当前主后端）
                    └──► AetherMind（后期备选后端）
                              └──► 业务 API（hdcrm 等）
```

IntentGate 与 Agent 平台**平级解耦**：换 Agent 框架不换通道；加通道不换 Agent。

---

## 3. 目标与非目标

### 3.1 目标

1. 定义并实现跨客户端统一的 **CardIntent 协议**
2. 提供 **Card Runtime**：YAML 模板 → 各端原生卡片
3. 实现 **快路径 / 慢路径** 事件路由，满足企微 5 秒更新等时限
4. **主接 AgentScope**，预留 AetherMind 同接口适配
5. 首个生产通道：**企业微信智能机器人长连接**

### 3.2 非目标（首版不做）

- 替代 AgentScope / AetherMind 的多 Agent 编排能力
- 自建 LLM 模型路由与微调
- 完整多租户 / RBAC（可后期在 Gateway 前加 API 网关）
- 飞书、钉钉全量生产上线（P3 验证抽象即可）

## 4. 用户与场景

| 角色 | 诉求 |
|------|------|
| 业务用户 | 在 IM 里审批、查单、确认告警，点击即反馈 |
| 开发（Agent） | 调用 `show_card` 工具即可，无需学企微 JSON |
| 开发（通道） | 实现 Channel Adapter，复用 Card Runtime |
| 运维 | Gateway 独立部署、可观测、断线自愈 |

### 优先场景（P1–P2）

1. **CRM 审批**：`button_interaction`，通过 / 驳回
2. **查询结果摘要**：`text_notice` + 详情链接
3. **系统告警**：`text_notice` + 确认按钮
4. **异步任务完成**：`aibot_send_msg` 主动推卡片

## 6. 技术选型

| 项 | 选型 | 理由 |
|----|------|------|
| 语言 | Python 3.11+ | 与 AgentScope 同栈 |
| Web 框架 | FastAPI | 异步、OpenAPI、团队熟悉 |
| 配置 | pydantic-settings | 类型安全 |
| 模板 | YAML + Jinja 式 `{{slot}}` | 可读、可版本管理 |
| 会话存储 | Redis（生产）/ 内存（开发） | 快路径 O(1) 查 task_id |
| Agent 通信 | HTTP + SSE | 流式 AgentReply |
| 企微连接 | websockets | 长连接 |

---

## 7. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 企微 5 秒 update 超时 | 卡片无法更新 | 快路径 + 占位卡；Agent 异步 |
| 单 Bot 单连接被踢 | 消息丢失 | 单进程部署 + 重连 + 告警 |
| 流式与卡片同包 | 企微拒收 | Gateway 强制分包 |
| Agent 响应 >30s | 用户无反馈 | 先 stream「思考中」 |
| 模板槽位缺失 | 渲染异常 | 启动时 Schema 校验 |

---

## 8. 成功标准

| 指标 | 目标 |
|------|------|
| 审批卡片端到端 | 用户发消息 → Agent 发卡 → 点击通过 → 5s 内 update |
| 快路径占比 | 标准按钮操作 >80% 不调 LLM |
| Gateway 可用性 | 断线 30s 内自动恢复 |
| 模板复用 | 同一 YAML 至少支持企微 + 一种其他端 |
| Agent 切换 | 改配置即可 mock → agentscope → aethermind |
