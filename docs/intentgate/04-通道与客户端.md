# 04 通道与客户端

Channel Adapter 只做**协议翻译**，卡片结构由 Card Runtime 统一生成。

---

## 1. Adapter 通用规范

### 1.1 生命周期

```
start()  → 建立连接 / 注册 webhook
loop     → 收消息 → normalize → 交 Gateway
stop()   → 关闭连接
```

### 1.2 入站映射

| 原生事件 | 归一化类型 |
|----------|------------|
| 用户文本消息 | `StandardMessage` |
| 卡片按钮/表单 | `CardActionEvent` |
| 进入会话 | 可选欢迎语（不送 Agent） |

### 1.3 出站映射

| AgentReply.kind | 通道行为 |
|-----------------|----------|
| `text` | 文本消息 |
| `stream` | 流式消息（若通道支持） |
| `card` | CardRuntime.render → 原生卡片 |
| `markdown` | Markdown 消息 |

### 1.4 通道能力矩阵

| 能力 | 企微 | 飞书 | 钉钉 | Web | API |
|------|------|------|------|-----|-----|
| 模板卡片 | ✅ | ✅ | ActionCard | React 组件 | JSON |
| 流式输出 | ✅ | 部分 | ❌ | SSE | SSE |
| 卡片 update | ✅ 5s | ✅ | 部分 | 即时 | N/A |
| 主动推送 | ✅ | ✅ | ✅ | WS 推送 | POST |
| 长连接 | ✅ WS | Webhook | Webhook | WS/SSE | — |

## 2. 企业微信（首个生产通道）

### 2.1 接入方式

使用**智能机器人长连接**（WebSocket），**不是**：

- 应用消息 `cgi-bin/message/send`
- 群机器人 Webhook（无会话交互）

| 特性 | Webhook | WebSocket 长连接 |
|------|---------|------------------|
| 连接 | 每次新建 | 复用 |
| 延迟 | 较高 | 低 |
| 公网 URL | 需要 | **不需要** |
| 加解密 | 需要 | **不需要** |
| 心跳 | 无 | 每 30 秒 `ping` |

- 连接地址：`wss://openws.work.weixin.qq.com`
- **每个 Bot 同一时间仅一个有效连接**；新 `aibot_subscribe` 会踢掉旧连接

### 2.2 前置配置

1. 管理后台：智能机器人 → **API 模式** → **长连接**
2. 获取 **BotID**、**Secret**（长连接专用，非回调 Token/EncodingAESKey）
3. 环境变量：`WECOM_BOT_ID`、`WECOM_BOT_SECRET`、`WECOM_WS_URL`

### 2.3 连接与鉴权

订阅：

```json
{
  "cmd": "aibot_subscribe",
  "headers": { "req_id": "REQUEST_ID" },
  "body": {
    "bot_id": "BOTID",
    "secret": "SECRET"
  }
}
```

保活：约每 **30 秒** `ping`，收到 `pong`。

### 2.4 交互流程

| 步骤 | 企微回调/命令 | IntentGate 行为 |
|------|---------------|-----------------|
| 连接 | `aibot_subscribe` | Adapter 鉴权 |
| 进入会话 | `aibot_event_callback`（`enter_chat`） | 可选 `aibot_respond_welcome_msg` |
| 用户消息 | `aibot_msg_callback` | → `StandardMessage` → Agent |
| 点按钮 | `template_card_event` | → `CardActionEvent` → 快/慢路径 |
| 回复 | `aibot_respond_msg` | ← AgentReply |
| 更新卡片 | `aibot_respond_update_msg` | ← 快路径 / 占位 |
| 主动推 | `aibot_send_msg` | ← 异步结果 |
| 心跳 | `ping` / `pong` | Adapter 定时 |

### 2.5 三种卡片出站场景

| 场景 | cmd | 触发 |
|------|-----|------|
| 回复用户 | `aibot_respond_msg` | 收到 `aibot_msg_callback` 后 |
| 主动推送 | `aibot_send_msg` | 告警、异步完成（用户须先在同会话发过消息） |
| 更新卡片 | `aibot_respond_update_msg` | 收到 `template_card_event` 后 |

### 2.6 频率与时限（必须遵守）

| 规则 | 值 |
|------|-----|
| 回复窗口 | 收到用户消息后 **24 小时内** |
| 会话限流 | **30 条/分钟**，**1000 条/小时** |
| 卡片 update | 点击后 **5 秒内** `aibot_respond_update_msg` |
| 流式 | 同一 `stream.id` 刷新，**10 分钟内** `finish=true` |
| 流式+卡片 | **不可同包**，须分两条 `aibot_respond_msg` |

### 2.7 回复消息示例

**模板卡片**（`req_id` 必须透传回调值）：

```json
{
  "cmd": "aibot_respond_msg",
  "headers": { "req_id": "来自 aibot_msg_callback 的 req_id" },
  "body": {
    "msgtype": "template_card",
    "template_card": {
      "card_type": "text_notice",
      "main_title": { "title": "工单提醒", "desc": "待审核" },
      "horizontal_content_list": [
        { "keyname": "单号", "value": "20260520001" }
      ]
    }
  }
}
```

**流式**：

```json
{
  "cmd": "aibot_respond_msg",
  "headers": { "req_id": "REQUEST_ID" },
  "body": {
    "msgtype": "stream",
    "stream": { "id": "STREAMID", "finish": false, "content": "正在查询..." }
  }
}
```

### 2.8 主动推送示例

```json
{
  "cmd": "aibot_send_msg",
  "headers": { "req_id": "自生成唯一 ID" },
  "body": {
    "chatid": "CHATID",
    "chat_type": 2,
    "msgtype": "template_card",
    "template_card": {
      "card_type": "button_interaction",
      "main_title": { "title": "系统告警", "desc": "CPU > 90%" },
      "button_list": [
        { "text": "确认", "style": 1, "key": "confirm" }
      ],
      "task_id": "alert:cpu:001:v1"
    }
  }
}
```

**chat_type**：1 单聊（chatid=userid），2 群聊，建议显式指定。

### 2.9 更新卡片示例

```json
{
  "cmd": "aibot_respond_update_msg",
  "headers": { "req_id": "来自事件回调的 req_id" },
  "body": {
    "response_type": "update_template_card",
    "template_card": {
      "card_type": "button_interaction",
      "main_title": { "title": "告警", "desc": "已确认处理中" },
      "button_list": [{ "text": "处理中", "style": 1, "key": "confirm" }],
      "task_id": "与发卡时一致的 TASK_ID"
    }
  }
}
```

### 2.10 WeComAdapter 实现要点（P1）

代码路径：`intentgate/adapters/wecom/`

| 任务 | 说明 |
|------|------|
| WS 客户端 | 连接、subscribe、ping 定时器 |
| 重连 | 指数退避，单进程单连接 |
| req_id | 入站存 Session，出站透传 |
| 归一化 | `aibot_msg_callback` → StandardMessage |
| 事件 | `template_card_event` → CardActionEvent |
| 出站 | AgentReply → 对应 cmd + body |
| 分包 | stream finish 后再发 card |

官方 API 文档：https://developer.work.weixin.qq.com/document/path/101463

SDK 参考：`wecom-aibot-python-sdk`

## 5. Web 通道（AetherMind UI）

### 5.1 场景

AetherMind 对话页内嵌卡片组件，与 IM 共用 `CardIntent`。

### 5.2 WebRenderer 输出

```json
{
  "component": "CardPanel",
  "template": "crm_approve",
  "props": {
    "title": "20260520001 待审批",
    "rows": [...],
    "actions": [{ "key": "approve", "label": "通过" }]
  },
  "task_id": "crm:approve:12345:v1"
}
```

前端 React 组件按 `template` 选择布局，点击回调 `POST /api/v1/actions`。
