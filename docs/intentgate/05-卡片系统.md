# 05 卡片系统

---

## 1. 设计理念

```
Agent 产出 CardIntent（模板名 + 槽位）
        ↓
Card Runtime 加载 cards/{template}.yaml
        ↓
填槽 {{variable}}
        ↓
CardRenderer 输出客户端原生 JSON
        ↓
Channel Adapter 发送
```

**禁止**：LLM 直接生成完整 `template_card` JSON。

---

## 2. CardIntent 与模板的关系

| 概念 | 说明 | 示例 |
|------|------|------|
| `CardIntent.template` | YAML **文件名**（不含扩展名） | `crm_approve` |
| YAML 内 `template` 字段 | 客户端**卡片类型** | `button_interaction` |
| `slots` | 运行时槽位值 | `{ "orderNo": "20260520001" }` |
| `task_id` | 卡片实例 ID | `crm:approve:12345:v1` |

---

## 3. YAML 模板规范

### 3.1 文件位置

```
practice/intentgate/cards/
├── crm_approve.yaml      # 审批
├── query_summary.yaml    # 查询摘要
├── alert_confirm.yaml    # 告警（待增）
└── ...
```

环境变量 `CARDS_DIR` 指向该目录。

### 3.2 字段定义

| 字段 | 必填 | 说明 |
|------|------|------|
| `template` | 是 | 客户端卡片类型（企微：text_notice / button_interaction / news_notice 等） |
| `slots` | 否 | 默认槽位或带 `{{var}}` 的标题模板 |
| `rows` | 否 | 横向内容列表 |
| `buttons` | 否 | 按钮列表 |
| `task_id` | 否 | task_id 表达式，支持 `{{orderId}}` |
| `version` | 否 | 模板版本（P2 Schema 校验） |

### 3.3 示例：crm_approve.yaml

```yaml
template: button_interaction
slots:
  title: "{{orderNo}} 待审批"
  desc: "{{amount}} 元 · {{shopName}}"
rows:
  - keyname: 申请人
    value: "{{applicant}}"
  - keyname: 单号
    value: "{{orderNo}}"
buttons:
  - key: approve
    text: 通过
    style: 1
  - key: reject
    text: 驳回
    style: 2
task_id: "crm:approve:{{orderId}}:v1"
```

### 3.4 示例：query_summary.yaml

```yaml
template: text_notice
slots:
  title: "{{title}}"
  desc: "{{desc}}"
rows:
  - keyname: "{{row1_key}}"
    value: "{{row1_val}}"
  - keyname: "{{row2_key}}"
    value: "{{row2_val}}"
```

### 3.5 槽位填充规则

- 语法：`{{variableName}}`，字母数字下划线
- 优先级：`CardIntent.slots` 覆盖 YAML 默认值
- 缺失：保留原 `{{var}}` 字符串（P2 改为校验报错）

## 4. Card Runtime

### 4.1 类职责

文件：`intentgate/runtime/card_runtime.py`

```python
class CardRuntime:
    def load_template(self, name: str) -> dict
    def render(self, channel: ChannelType, intent: CardIntent) -> dict
```

流程：

1. `load_template(intent.template)` → 读 YAML
2. 选择 `channel` 对应 Renderer
3. `renderer.render(intent, template_def)` → 原生 dict

### 4.2 调用方式

**HTTP 调试**：

```bash
curl -X POST http://127.0.0.1:8090/api/v1/cards/render \
  -H "Content-Type: application/json" \
  -d '{
    "channel": "wecom",
    "intent": {
      "template": "crm_approve",
      "slots": {
        "orderNo": "20260520001",
        "orderId": "12345",
        "amount": "9800",
        "shopName": "华东店",
        "applicant": "张三"
      }
    }
  }'
```

**代码内**：

```python
runtime = CardRuntime(Path("./cards"))
body = runtime.render(ChannelType.WECOM, intent)
# body 可直接作为 aibot_respond_msg 的 body 部分
```

## 6. task_id 与状态机

### 6.1 命名规范

```
{业务域}:{动作}:{业务ID}:{版本}
```

示例：

- `crm:approve:12345:v1`
- `alert:cpu:host-01:v1`
- `todo:item:9876:v2`（改单升版本）

### 6.2 状态流转

```
pending ──approve──► approved
        ──reject───► rejected
        ──confirm──► confirmed
        ──cancel───► cancelled
```

状态存于 Session Store `active_cards[task_id].state`。

### 6.3 快路径 handler 伪代码

```python
async def handle_approve(event: CardActionEvent):
    card = session_store.get_card(event.task_id)
    if card["state"] != "pending":
        return render_already_done(card)

    await crm_api.approve(biz_id=parse_biz_id(event.task_id))
    session_store.update_card_state(event.task_id, "approved")
    return runtime.render(channel, CardIntent(
        template=card["template"],
        slots={**card["slots"], "title": "已通过", "desc": "审批完成"},
    ))
```

## 7. 优先实现的四类卡片

| 场景 | 模板名 | 企微 card_type | 按钮 |
|------|--------|----------------|------|
| 待办/审批 | `crm_approve` | button_interaction | 通过 / 驳回 |
| 查询摘要 | `query_summary` | text_notice | 跳转链接（P2） |
| 系统告警 | `alert_confirm` | button_interaction | 确认 / 误报 |
| 选项菜单 | `option_menu` | multiple_interaction | 多选项（P2） |

## 8. 模板版本管理（P2）

```
cards/
├── crm_approve.yaml
├── crm_approve.v2.yaml
└── schema/
    └── crm_approve.schema.json
```

- 启动时校验 slots 必填项
- `task_id` 版本段与 YAML 版本对齐
- 变更模板时升 `v2`，旧卡片仍用旧 task_id

## 10. 反模式

| 反模式 | 正确 |
|--------|------|
| Agent 输出 template_card JSON | CardIntent + 模板 |
| 模板名与 card_type 混淆 | template=文件名，YAML.template=card_type |
| 无 task_id 发 button 卡 | 必须带 task_id |
| 同一业务复用 task_id | 改单升版本号 |
