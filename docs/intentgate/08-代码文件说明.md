# 08 代码文件说明与阅读顺序

**代码根目录**：`practice/intentgate/`

本文为阅读代码时的**索引**：每个文件的一句话职责，以及推荐的阅读顺序（与 [`03-协议规范`](./03-协议规范.md)、[`07-实施与运维`](./07-实施与运维.md) 互为补充：`07` 侧重目录树与运维；本文侧重「从哪一行开始读」）。

## 1. 推荐阅读顺序（总览）

按数据流从上到下读一遍，能快速串起整张图：**协议 → Gateway API → Runtime 渲染 → 适配器后端**。

| 顺序 | 路径 | 说明 |
|------|------|------|
| ① | [`practice/intentgate/pyproject.toml`](../../practice/intentgate/pyproject.toml) | 包名、依赖、可编辑安装方式 |
| ② | [`practice/intentgate/.env.example`](../../practice/intentgate/.env.example) | 环境变量与默认值（对齐 `intentgate/config.py`） |
| ③ | [`intentgate/config.py`](../../practice/intentgate/intentgate/config.py) | `Settings`、`CARDS_PATH`、Agent 后端选择 |
| ④ | [`intentgate/schemas/message.py`](../../practice/intentgate/intentgate/schemas/message.py) | `StandardMessage`、`CardActionEvent`、`ChannelType`（入站） |
| ⑤ | [`intentgate/schemas/reply.py`](../../practice/intentgate/intentgate/schemas/reply.py) | `AgentReply`、`CardIntent`、卡片动作结构（出站） |
| ⑥ | [`intentgate/schemas/__init__.py`](../../practice/intentgate/intentgate/schemas/__init__.py) | 对外 re-export |
| ⑦ | [`intentgate/adapters/agent_backend.py`](../../practice/intentgate/intentgate/adapters/agent_backend.py) | `AgentBackend` 抽象：chat / handle_action |
| ⑧ | [`intentgate/runtime/renderers/base.py`](../../practice/intentgate/intentgate/runtime/renderers/base.py) | `CardRenderer` 协议 |
| ⑨ | [`intentgate/runtime/renderers/wecom.py`](../../practice/intentgate/intentgate/runtime/renderers/wecom.py) | 企微通道渲染实现 |
| ⑩ | [`intentgate/runtime/card_runtime.py`](../../practice/intentgate/intentgate/runtime/card_runtime.py) | YAML 加载 + 按通道分发 Renderer |
| ⑪ | [`intentgate/gateway/api.py`](../../practice/intentgate/intentgate/gateway/api.py) | `/api/v1`：messages、actions、cards/render、protocol |
| ⑫ | [`intentgate/gateway/router.py`](../../practice/intentgate/intentgate/gateway/router.py) | 快路径 action key 常量与判定（供适配层引用） |
| ⑬ | [`intentgate/gateway/session.py`](../../practice/intentgate/intentgate/gateway/session.py) | 内存会话存储（生产可换 Redis） |
| ⑭ | [`intentgate/adapters/mock.py`](../../practice/intentgate/intentgate/adapters/mock.py) | 本地假 Agent，便于不调后端跑通链路 |
| ⑮ | [`intentgate/adapters/agentscope.py`](../../practice/intentgate/intentgate/adapters/agentscope.py) | AgentScope HTTP/SSE 对接骨架 |
| ⑯ | [`intentgate/adapters/aethermind.py`](../../practice/intentgate/intentgate/adapters/aethermind.py) | AetherMind 对接骨架 |
| ⑰ | [`intentgate/adapters/__init__.py`](../../practice/intentgate/intentgate/adapters/__init__.py) | `create_agent_backend(settings)` 工厂 |
| ⑱ | [`intentgate/adapters/wecom/__init__.py`](../../practice/intentgate/intentgate/adapters/wecom/__init__.py) | 企微长连接适配占位（P1） |
| ⑲ | [`intentgate/main.py`](../../practice/intentgate/intentgate/main.py) | FastAPI 应用、lifespan、全局 getter |
| ⑳ | [`cards/*.yaml`](../../practice/intentgate/cards/) | 卡片模板示例（与 [`05-卡片系统`](./05-卡片系统.md) 对照） |
| ㉑ | [`tests/test_card_runtime.py`](../../practice/intentgate/tests/test_card_runtime.py) | CardRuntime / 渲染相关单测 |

**捷径**：若只想跑通 HTTP，可读 **③ → ④⑤ → ⑩⑪ → ⑲**，其余按需深入。

## 2. 按目标的阅读顺序

| 目标 | 建议顺序 |
|------|----------|
| 理解 CardIntent 与 API 契约 | ④⑤ → ⑪ `protocol` 端点 → 对照 [`03-协议规范`](./03-协议规范.md) |

| 快路径按钮逻辑 | ⑫ → 架构文档 [`02`](./02-架构设计.md) 中「快/慢路径」 |

## 3. 文件清单（逐项说明）

### 3.1 工程根（`practice/intentgate/`）

| 文件 | 说明 |
|------|------|
| `README.md` | 启动命令与健康检查；链到文档中心 |
| `pyproject.toml` | 包元数据与 setuptools 入口 |
| `requirements.txt` | 依赖列表（与 pyproject 二选一安装时可参考） |
| `.env.example` | 可复制为 `.env` 的配置模板 |
| `.gitignore` | 忽略 `.venv`、`__pycache__`、`.env` 等 |

### 3.2 卡片模板（`cards/`）

| 文件 | 说明 |
|------|------|
| `crm_approve.yaml` | 审批类卡片模板示例 |
| `query_summary.yaml` | 查询摘要类卡片模板示例 |

### 3.3 包 `intentgate/`（应用代码）

| 文件 | 说明 |
|------|------|
| `__init__.py` | 包标识 |
| `main.py` | FastAPI App、lifespan 初始化后端与 CardRuntime、`/health`、对外 getter |
| `config.py` | Pydantic Settings：端口、后端类型、卡片目录、会话存储等 |

**`schemas/`**

| 文件 | 说明 |
|------|------|
| `message.py` | 用户文本/卡片动作的入站模型与通道枚举 |
| `reply.py` | Agent 回复与 `CardIntent` 等与 LLM/Gateway 约定字段 |
| `__init__.py` | 常用类型聚合导出 |

**`gateway/`**

| 文件 | 说明 |
|------|------|
| `api.py` | Gateway HTTP 路由：消息入站、动作入站、渲染、JSON Schema 协议快照 |
| `router.py` | `is_fast_path` 与预设 action_key 集合 |
| `session.py` | 进程内会话表（可按 `SESSION_STORE` 扩展） |

**`runtime/`**

| 文件 | 说明 |
|------|------|
| `card_runtime.py` | 读取 YAML、`CardIntent` → 原生 dict |
| `renderers/base.py` | 渲染器抽象 |
| `renderers/wecom.py` | 企微格式渲染 |

**`adapters/`**

| 文件 | 说明 |
|------|------|
| `agent_backend.py` | 异步迭代 `AgentReply` 的抽象基类 |
| `mock.py` | 返回固定/简单回复的 Mock 实现 |
| `agentscope.py` | AgentScope 后端占位与 HTTP 客户端形态 |
| `aethermind.py` | AetherMind 后端占位 |
| `__init__.py` | 根据 `AGENT_BACKEND` 选择具体 Backend |
| `wecom/__init__.py` | 企微 WebSocket 等企业对接占位 |

### 3.4 测试（`tests/`）

| 文件 | 说明 |
|------|------|
| `test_card_runtime.py` | 覆盖模板加载与渲染行为（回归防改坏 YAML/Renderer） |
