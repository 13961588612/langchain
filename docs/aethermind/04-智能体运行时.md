# 04 智能体运行时

> **演进方向**（Phase 2）：当前 `AgentRuntime` 将重构为 **AgentLoop** 体系中的 `DeepAgentsBackend`。详见 [02-架构设计](./02-架构设计.md#4-agent-loop-抽象层规划phase-2) 与 [09-对标Hermes与规划修订](./09-对标Hermes与规划修订.md)。

## 1. 概述

`AgentRuntime`（`backend/app/core/agent_runtime.py`）是 AetherMind 的**当前**执行引擎，负责：

- 从 DB 配置构建 DeepAgent
- LangGraph Checkpointer 多轮持久化
- SSE 流式输出（token、tool_call、tool_result）
- 工作目录与 Skills 加载
- Soul / Profile 注入 system prompt

全局单例：`agent_runtime = AgentRuntime()`

---

## 2. 构建流程

```python
create_deep_agent(
    model=model,              # ModelHub
    system_prompt=full_prompt,# base + soul + profile
    backend=FilesystemBackend(root_dir=work_dir),
    skills=[skills_dir],      # workdirs/{id}/skills/
    checkpointer=AsyncPostgresSaver,
)
```

### 2.1 配置来源

| 配置项 | 来源 |
|--------|------|
| provider / model_name | `Agent` 表 |
| model_parameters | JSON 字符串解析 |
| system_prompt | `Agent.system_prompt` |
| soul_config | `Agent.soul_config` 或 soul.md（Phase 4） |
| profile_config | `Agent.profile_config` 或 profile.md |
| work_directory | `Agent.work_directory` 或 `WORKDIR_ROOT/{id}` |

### 2.2 System Prompt 组装

```python
def _build_system_prompt(base, soul_config, profile_config):
    parts = [base]
    if soul_config:
        parts.append(f"\n## Personality & Soul\n{soul_config}")
    if profile_config:
        parts.append(f"\n## Behavior Profile\n{profile_config}")
    return "\n\n".join(parts)
```

Phase 2 将升级为独立 `PromptAssembler` 模块，支持 soul.md YAML frontmatter。

## 3. 对话执行

### 3.1 stream_chat

```python
async def stream_chat(
    agent_id: str,
    thread_id: str,
    message: str,
    agent_config: dict | None = None,
) -> AsyncGenerator[dict, None]:
```

输入：

```python
input_state = {"messages": [{"role": "user", "content": message}]}
config = {"configurable": {"thread_id": thread_id}}
```

通过 `agent.astream_events(..., version="v2")` 监听：

| LangGraph 事件 | 映射输出 |
|----------------|----------|
| `on_chat_model_stream` | `event=token` |
| `on_tool_start` | `event=tool_call` |
| `on_tool_end` | `event=tool_result` |
| 结束 | `event=done` |
| 异常 | `event=error` |

### 3.2 Agent 缓存

- Key：`agent_id`
- 首次 `get_or_create_agent` 构建并缓存
- `clear_agent_cache(agent_id)` 在配置/Skill 变更后调用

## 4. Checkpointer（Layer 1 记忆）

使用 `langgraph.checkpoint.postgres.aio.AsyncPostgresSaver`：

- 连接串：`DATABASE_URL`（asyncpg → psycopg 格式转换）
- `thread_id` = `Conversation.thread_id`
- 存储最近对话图状态，支持多轮工具调用

Phase 4 补充 Layer 2（摘要）与 Layer 3（pgvector 长期记忆）。

## 8. 卡片工具（IntentGate 集成，Phase 9）

Agent 需注册 `show_card` 工具，返回 IntentGate 约定的 `card_intent`：

```python
def show_card(template: str, slots: dict, task_id: str = None) -> str:
    import json
    return json.dumps({
        "type": "card_intent",
        "template": template,
        "slots": slots,
        "task_id": task_id,
    })
```

`stream_chat` 扩展：检测 tool 输出含 `card_intent` 时 yield `event=card_intent`，供 IntentGate 消费。
