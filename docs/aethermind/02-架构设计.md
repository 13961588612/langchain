## 3. 核心数据流：Web 对话

```
用户在前端输入消息
    → POST /api/conversations/{id}/chat  {"message": "..."}
    → 保存 user Message 到 PostgreSQL
    → AgentRuntime.stream_chat(agent_id, thread_id, message)
         → get_or_create_agent (DeepAgents + Checkpointer)
         → astream_events → token / tool_call / tool_result
    → SSE 推送到前端
    → done 时保存 assistant Message
```

`thread_id` 存于 `Conversation` 表，对应 LangGraph Checkpointer 的会话键，实现多轮上下文。

## 4. Agent Loop 抽象层（规划，Phase 2）

对标 Hermes `AIAgent`：**平台无关的一轮对话执行契约**，多入口复用同一内核。

```
Web SSE ──────┐
IntentGate ───┼──► AgentLoop.run_turn(ctx, message) ──► AsyncIterator[AgentEvent]
Scheduler ────┤              │
Batch/API ────┘              ▼
                    DeepAgentsBackend（当前 AgentRuntime 演进）
                    （未来可增 AgentScopeBackend 等）
```

### 4.1 接口（规划）

```python
class AgentLoopBackend(ABC):
    async def run_turn(
        self,
        ctx: TurnContext,  # agent_id, thread_id, channel, user_id
        message: str,
        *,
        cancel_token: CancelToken | None = None,
    ) -> AsyncIterator[AgentEvent]:
        """事件：token | tool_call | tool_result | card_intent | done | error"""
```

### 4.2 设计原则

| 原则 | 说明 |
|------|------|
| Prompt 稳定性 | 对话中不随意改 system prompt |
| 可中断 | `cancel_token` 取消进行中的 turn（Phase 3） |
| 可观测 | 每 turn 生成 `turn_id`，关联 LangSmith |

## 7. Tool Registry（规划，Phase 2）

中央工具注册与 **toolsets** 管理，对标 Hermes `tools/registry.py` + toolsets。

```
tools.yaml / Agent 配置
    → ToolRegistry.resolve(agent_id, channel)
    → enabled: [filesystem, mcp, crm, card]
    → disabled: [terminal]  # 默认关闭
    → guard: write_file → require_approval
    → DeepAgents / MCP 工具注入
```

| Toolset | 说明 | 默认 |
|---------|------|------|
| `filesystem` | read/write/list | 开发 Agent |
| `mcp` | MCP 动态工具 | Phase 2 |
| `card` | show_card（IntentGate） | IM Agent |
| `crm` | 业务 API | 业务 Agent |
| `terminal` | execute_command | **关闭** |
| `delegate` | spawn_subagent | Phase 6 |

---

## 8. ToolGuard — 最小 HITL（规划，Phase 4）

在完整审批 UI（Phase 7）之前，工具层拦截危险操作：

```
tool_call 请求
    → ToolGuard.check(tool_name, args)
    → deny | require_approval | allow
    → deny：返回错误给 Agent
    → require_approval：Phase 4 同步拒绝并提示；Phase 7 走审批流
```

默认：`execute_command` **deny**；`write_file` **require_approval**。

---

## 9. 任务队列与 Scheduler（规划，Phase 3 / 8）

**Phase 3 — Cancel / Queue**

- Redis 队列：长任务（>30s）异步执行
- `POST /chat` 返回 task_id；`DELETE /tasks/{id}` 取消
- SSE 仍用于短对话

**Phase 8 — Scheduler**

```
Cron Job → AgentLoop.run_turn → card_intent/text → IntentGate 主动推送
```

## 10. 与 IntentGate 的协作架构

```
企微用户
    → IntentGate WeComAdapter
    → (快路径) Card Runtime 直接 update
    → (慢路径) AetherMindBackend
         → POST /api/agents/{id}/chat  (SSE)
         → Agent 调 show_card 工具
         → SSE 返回 card_intent 事件
    → IntentGate 渲染 template_card → 企微
```

AetherMind **不处理**企微 `aibot_*` 协议；只提供标准 Agent 推理与 `CardIntent` 产出（P9 扩展 SSE）。

        ↓

不在 AetherMind 内重复实现 Channel Gateway，而是：

- Web：原生 SSE（已实现）
- IM：**IntentGate** 作为唯一 IM 网关

`Channel` ORM 模型保留，用于配置 IntentGate 路由到哪个 `agent_id`。

## 13. 安全边界

| 边界 | 机制 |
|------|------|
| API 访问 | API Key / JWT（待完善） |
| Agent 文件 | FilesystemBackend 限制在 workdir |
| 敏感工具 | Phase 4 ToolGuard + Phase 7 HITL |
| LLM Key | 环境变量 + ModelConfig 加密（Phase 3） |
| IM 入口 | IntentGate 隔离，不暴露 AetherMind 到公网 |

## 14. 扩展原则

1. **Agent 逻辑在 AgentLoop/Core**，API 层薄封装
2. **配置进 DB + workdir 文件**，避免硬编码
3. **IM 走 IntentGate**，AetherMind 只扩展 SSE 事件类型
4. **Phase 1.5 门禁**后再叠功能；详见 [09-对标Hermes与规划修订](./09-对标Hermes与规划修订.md)
