# 02 架构设计

## 1. 总体架构

```
┌──────────────────────────────────────────────────────────────────┐
│                         客户端层                                  │
│   Next.js Web UI  │  REST 调用方  │  IntentGate (IM)             │
└─────────┬─────────────────┬──────────────────┬─────────────────┘
          │                 │                  │
          ▼                 ▼                  ▼
┌──────────────────────────────────────────────────────────────────┐
│                    AetherMind Backend (FastAPI)                   │
│  ┌─────────────┐  ┌──────────────┐  ┌──────────────────────────┐ │
│  │ API Layer   │  │ AgentRuntime │  │ ModelHub                 │ │
│  │ agents/     │  │ DeepAgents   │  │ init_chat_model          │ │
│  │ conversations│ │ LangGraph    │  │ 5 Providers              │ │
│  └─────────────┘  └──────────────┘  └──────────────────────────┘ │
│  ┌─────────────┐  ┌──────────────┐  ┌──────────────────────────┐ │
│  │ ORM Models  │  │ Middleware   │  │ Workdirs (Filesystem)    │ │
│  │ PostgreSQL  │  │ Logging/Token│  │ skills/ agent.md ...     │ │
```

## 2. 分层职责

| 层 | 路径 | 职责 |
|----|------|------|
| **前端** | `frontend/src/` | Agent 管理、对话 UI、后续 Soul/Workdir 编辑器 |
| **API** | `backend/app/api/` | REST + SSE 端点 |
| **Schema** | `backend/app/schemas/` | Pydantic 请求/响应模型 |
| **Model** | `backend/app/models/` | SQLAlchemy ORM |
| **Core** | `backend/app/core/` | AgentRuntime、ModelHub |
| **Middleware** | `backend/app/middleware/` | 请求日志、Token 统计 |
| **Workdirs** | `workdirs/{agent_id}/` | Agent 文件系统工作区 |

## 3. 核心数据流：Web 对话

```
用户在前端输入消息
    → POST /api/conversations/{id}/chat  {"message": "..."}
    → 保存 user Message 到 PostgreSQL
    → AgentRuntime.stream_chat(agent_id, thread_id, message)
         → get_or_create_agent (DeepAgents + Checkpointer)
         → astream_events → token / tool_call / tool_result
    → SSE 推送到前端
    → done 时保存 assistant Message
```

`thread_id` 存于 `Conversation` 表，对应 LangGraph Checkpointer 的会话键，实现多轮上下文。

---

## 4. AgentRuntime 架构

```
Agent 配置 (DB)
    │
    ├── ModelHub.get_model(provider, model_name)
    ├── FilesystemBackend(workdirs/{agent_id})
    ├── skills/ 目录
    ├── system_prompt + soul_config + profile_config
    └── AsyncPostgresSaver (Checkpointer)
            │
            ▼
    create_deep_agent(...) → CompiledStateGraph
            │
            ▼
    astream_events(input_state, thread_id)
```

Agent 实例按 `agent_id` **缓存**；配置变更后需 `clear_agent_cache(agent_id)`。

---

## 5. 与 IntentGate 的协作架构

```
企微用户
    → IntentGate WeComAdapter
    → (快路径) Card Runtime 直接 update
    → (慢路径) AetherMindBackend
         → POST /api/agents/{id}/chat  (SSE)
         → Agent 调 show_card 工具
         → SSE 返回 card_intent 事件
    → IntentGate 渲染 template_card → 企微
```

AetherMind **不处理**企微 `aibot_*` 协议；只提供标准 Agent 推理与 `CardIntent` 产出（P9 扩展 SSE）。

## 6. 未来模块架构（Phase 2–9）

### 6.1 Prompt Assembler（Phase 2）

```
soul.md + profile.md + base system_prompt + tools.yaml + skills/
                    ↓
            Prompt Assembler
                    ↓
            create_deep_agent(system_prompt=...)
```

### 6.2 Model Hub 增强（Phase 3）

```
Agent.model_provider / model_name
        ↓
ModelConfig 表（加密 API Key、路由策略）
        ↓
ModelHub.router (fixed / fallback_chain / cheapest_first)
        ↓
init_chat_model
```

### 6.3 工作目录（Phase 4）

```
workdirs/{agent_id}/
├── agent.md
├── soul.md
├── profile.md
├── memory.md
├── tools.yaml
├── skills/
└── .versions/
```

前端 IDE 编辑 → 保存文件 → `clear_agent_cache` → 下次对话生效。

### 6.4 编排引擎（Phase 6）

```
orchestrations/*.yaml
        ↓
OrchestrationBuilder → LangGraph StateGraph (Supervisor)
        ↓
OrchestrationRuntime → SSE 事件流
```

### 6.5 通道（Phase 9）

不在 AetherMind 内重复实现 Channel Gateway，而是：

- Web：原生 SSE（已实现）
- IM：**IntentGate** 作为唯一 IM 网关

`Channel` ORM 模型保留，用于配置 IntentGate 路由到哪个 `agent_id`。

## 8. 安全边界

| 边界 | 机制 |
|------|------|
| API 访问 | API Key / JWT（待完善） |
| Agent 文件 | FilesystemBackend 限制在 workdir |
| 敏感工具 | HITL 审批（Phase 7） |
| LLM Key | 环境变量 + ModelConfig 加密（Phase 3） |
| IM 入口 | IntentGate 隔离，不暴露 AetherMind 到公网 |

---

## 9. 扩展原则

1. **Agent 逻辑在 Core**，API 层薄封装
2. **配置进 DB + workdir 文件**，避免硬编码
3. **IM 走 IntentGate**，AetherMind 只扩展 SSE 事件类型
4. **Phase 按需增量**，不提前实现企业级复杂度
