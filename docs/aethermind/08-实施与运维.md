## 3. 快速启动（Docker）

```bash
cd practice/aethermind
cp .env.example .env
# 编辑 .env，至少配置 OPENAI_API_KEY

docker-compose up -d
```

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:3000 |
| 后端 API | http://localhost:8000 |
| Swagger | http://localhost:8000/docs |
| 健康检查 | http://localhost:8000/api/health |
| PostgreSQL | localhost:5432 |
| Redis | localhost:6379 |

## 4. 环境变量

| 变量 | 说明 | 必填 |
|------|------|------|
| `APP_ENV` | development / production | — |
| `APP_SECRET_KEY` | 应用密钥 | 生产必填 |
| `DATABASE_URL` | asyncpg 连接串 | ✅ |
| `REDIS_URL` | Redis | ✅ |
| `OPENAI_API_KEY` | OpenAI | 至少一个 Provider |
| `ANTHROPIC_API_KEY` | Anthropic | 可选 |
| `GOOGLE_API_KEY` | Google Gemini | 可选 |
| `AZURE_OPENAI_*` | Azure | 可选 |
| `LANGSMITH_API_KEY` | LangSmith 追踪 | 推荐 |
| `LANGSMITH_PROJECT` | 项目名 | 默认 aethermind |
| `WORKDIR_ROOT` | 工作目录根 | 默认 backend/workdirs |

通道相关（Phase 9 / IntentGate）：

| 变量 | 说明 |
|------|------|
| `WECOM_*` | 企微（IntentGate 侧为主） |
| `FEISHU_*` | 飞书 |
| `DINGTALK_*` | 钉钉 |

## 5. 本地开发（非 Docker）

### 后端

```bash
cd practice/aethermind/backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
# 确保 PostgreSQL、Redis 已运行
uvicorn app.main:app --reload --port 8000
```

### 前端

```bash
cd practice/aethermind/frontend
npm install
npm run dev
```

`next.config.js` 将 `/api/*` 代理到 `backend:8000`。

## 7. 常用操作

### 创建 Agent 并联调

1. 打开 http://localhost:3000 → 创建 Agent
2. 填写 model_provider、model_name、system_prompt
3. 进入 Agent 详情 → 开始对话
4. 或 Swagger `POST /api/conversations` + `POST .../chat`

### 清除 Agent 缓存

配置变更后（代码或 API）：

```python
from app.core.agent_runtime import agent_runtime
agent_runtime.clear_agent_cache("agent-uuid")
```

（管理 API 待补）

### 查看日志

```bash
docker-compose logs -f backend
docker-compose logs -f frontend
```

## 9. 与 IntentGate 联调

1. 启动 AetherMind：`docker-compose up -d`（:8000）
2. 启动 IntentGate：`uvicorn intentgate.main:app --port 8090`
3. IntentGate `.env`：

```env
AGENT_BACKEND=aethermind
AGENT_BACKEND_URL=http://127.0.0.1:8000
AETHERMIND_AGENT_ID=<your-agent-uuid>
```

## 10. 故障排查

| 现象 | 排查 |
|------|------|
| 对话无响应 | Provider API Key、Agent is_active |
| 500 on chat | 后端日志、Postgres 连接、Checkpointer setup |
| 前端 API 404 | backend 是否启动、next 代理配置 |
| 工具不加载 | workdirs/{id}/skills/ 是否存在 |
| Token 不流式 | 浏览器 Network SSE、模型是否支持 stream |
