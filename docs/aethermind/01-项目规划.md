# 01 项目规划

## 1. 项目定位

AetherMind 是**公司内部智能体管理开发平台**（Agent Factory），不是面向公众的 SaaS 产品。

```
┌─────────────────────────────────────────────────┐
│              AetherMind (Internal)               │
│                                                  │
│  AI 开发团队 (Developer)                         │
│  · 创建/配置智能体                                │
│  · 编写 soul.md / profile.md / agent.md          │
│  · 开发/安装 Skills                               │
│  · 管理模型、编排多 Agent、配置审批                 │
│                                                  │
│  业务团队 (User)                                  │
│  · Web UI 与智能体对话                            │
│  · 通过 IM（经 IntentGate）使用智能体              │
│                                                  │
│  管理员 (Admin) — 简化版                          │
│  · 用户管理、用量监控、健康检查                     │
└─────────────────────────────────────────────────┘
```

**核心逻辑**：先让智能体「活起来」，再「强起来」，最后「管起来」。

---

## 2. 目标与非目标

### 2.1 目标

1. 提供 Agent CRUD、对话、SSE 流式输出的可运行平台（Phase 1 ✅）
2. 支持独立人格（Soul）与行为配置（Profile）
3. 统一 Model Hub，屏蔽多 Provider 差异
4. 工作目录 IDE：agent.md、soul.md、skills/、memory
5. 内部 Skills 开发、安装与共享
6. Supervisor 模式多智能体编排
7. 敏感操作 Human-in-the-Loop 审批
8. LangSmith 追踪 + 监控面板
9. 经 **IntentGate** 接入企微/飞书/钉钉（卡片交互）

### 2.2 非目标（内部版首阶段不做）

| 不做 | 原因 |
|------|------|
| 多租户隔离 | 单公司使用 |
| 复杂计费 | 内部结算即可 |
| Skills 公开市场 | 内部库足够 |
| OAuth2 完整自建 | 简化 API Key + JWT |
| Vault 密钥管理 | `.env` + 加密列（后期升级） |
| Feature Flags 灰度 | 内部工具不需要 |
| 多节点高可用 | 单机/小规模先行 |
| GDPR 专项合规 | 内部工具 |

企业化能力见 [07-企业加固规划](./07-企业加固规划.md)，**延后独立批次**实施。

### 排序原则

1. **Phase 1**：最小可运行闭环（已完成）
2. **Phase 2–4**：单个 Agent「更好用」（身份、模型、工作区）
3. **Phase 5–7**：平台「更强」（Skills、编排、审批）
4. **Phase 8**：可观测
5. **Phase 9**：IM 通道（依赖 IntentGate，与核心能力解耦）

## 5. Phase 1 交付摘要（已完成）

## 7. 与 IntentGate 的分工

| 系统 | 职责 |
|------|------|
| **AetherMind** | Agent 生命周期、推理、工具、记忆、Web UI |
| **IntentGate** | IM 协议、卡片渲染、快/慢路径、session/task_id |

IM 卡片不直接在 AetherMind 内实现 WebSocket，而是通过 IntentGate 的 `AetherMindBackend` 调用 AetherMind SSE API。详见 [06-IntentGate集成](./06-IntentGate集成.md)。

## 9. 风险与对策

| 风险 | 对策 |
|------|------|
| DeepAgents/LangGraph 版本升级 | 锁定 requirements，集成测试 |
| Token 成本失控 | Model Hub 路由 + 用量统计 + 告警 |
| Agent 工具误操作 | HITL + 工作目录沙箱 |
| IM 5s 卡片超时 | IntentGate 快路径，不经过 AetherMind |
| 单点部署故障 | 企业加固批次做 HA |
