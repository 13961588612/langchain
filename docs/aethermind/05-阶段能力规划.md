### 2.1 soul.md 格式

```markdown
---
name: "客服助手小A"
role: customer_support

  - "偶尔用生活化比喻"
```

## Phase 3：Model Hub 模型中心 + Cancel/Queue

**目标**：统一管理公司 LLM 资源，Agent 选模型不关心 Provider 细节。

### 3.1 管理界面

- Provider 列表：OpenAI / Anthropic / Google / Azure / Ollama
- 每 Provider 下模型列表、连接状态、API Key 配置
- 「扫描本地模型」：Ollama / vLLM 端点
- 模型标签：推荐 / 快速 / 推理 / 编程 / 长文本

### 3.2 ModelConfig 能力

- CRUD API + 前端配置页
- API Key AES 加密存储
- `POST /api/models/{id}/test` 连接测试
- 默认参数：temperature、max_tokens、top_p

### 3.3 路由策略（Agent 级）

| 策略 | 行为 |
|------|------|
| `fixed` | 固定指定模型 |
| `cheapest_first` | 优先便宜模型 |
| `best_quality` | 默认最强模型 |
| `fallback_chain` | 主模型失败依次降级 |

示例 fallback：

```yaml
primary: gpt-4o
fallback_1: claude-sonnet
fallback_2: qwen2.5:72b
final: gpt-4o-mini
```

- [ ] 路由策略执行器

## Phase 4：工作目录系统 + ToolGuard

**目标**：可编辑、可版本化的 Agent 工作区，作为「智能体开发 IDE」核心。

### 4.1 目录结构

```
workdirs/{agent_id}/
├── agent.md           # 智能体定义
├── soul.md
├── profile.md
├── memory.md          # 长期记忆快照
├── tools.yaml         # 工具声明
├── skills/
├── data/
│   ├── knowledge/
│   └── cache/
└── .versions/
    ├── agent.md.v1
    └── manifest.json
```

### 4.2 前端 IDE

- 左文件树 + 右编辑区
- YAML frontmatter 语法高亮与校验
- tools.yaml JSON Schema 补全
- 版本历史：diff、回滚、注释

### 4.3 Memory 三层

| 层 | 机制 | Phase |
|----|------|-------|
| L1 短期 | PostgresSaver Checkpointer | ✅ |
| L2 中期 | 每 10 轮或 >4000 tokens LLM 摘要 | Phase 4 |
| L3 长期 | pgvector 向量检索 + memory.md 快照 | Phase 4 |

> L1 压缩由 Phase 2 ContextEngine 负责；L2/L3 在本 Phase 完成。

前端 Memory 管理：查看/编辑/删除/标记重要度。

### 4.4 ToolGuard（最小 HITL）

**在 Phase 7 完整审批 UI 之前**，工具层必须拦截危险操作：

```
write_file      → require_approval（Phase 4 可先硬拒绝 + 提示）
execute_command → deny（默认）
delete_file     → require_approval
```

实现：`backend/app/core/tool_guard.py`，挂接 ToolRegistry dispatch。

### 4.5 tools.yaml

声明式工具配置 + 允许/禁止列表，与 DeepAgents 工具注册联动。

## Phase 5：Skills 开发工场

**目标**：团队开发、测试、共享内部 Skills。

### 5.1 定位

- **内部库**，非公开市场
- 流程：创建 → 编写 SKILL.md → 安装到 Agent 测试 → 发布到内部库

### 5.2 SKILL.md 规范

遵循 DeepAgents / Anthropic Agent Skill 约定：frontmatter（名称、描述、触发词）+ Markdown 正文。

### 5.3 创建方式

1. 前端 Skill 编辑器（表单 + Markdown）
2. 直接放入 `workdirs/{id}/skills/`
3. 从内部库一键安装

### 5.4 运行时

- 触发匹配 → 加载 Skill 注入上下文
- 来源优先级：Agent skills/ 目录
- 热加载：`clear_agent_cache(agent_id)`

## Phase 6：多智能体编排

**目标**：Supervisor 模式多 Agent 协作（优先覆盖 80% 场景）。

### 6.1 编排模式路线图

| 批次 | 模式 | 说明 |
|------|------|------|
| 1 | Supervisor | 分发 + 汇总 |
| 2 | Swarm / Handoff | Agent 间移交 |
| 3 | Map-Reduce | 并行子任务 |

### 6.2 YAML 配置示例

```yaml
name: "代码审查团队"
mode: supervisor

supervisor:
  agent: "tech-lead"
  instructions: "分析变更，分发给审查者"

members:
  - agent: "code-style-checker"
    role: "代码风格"
  - agent: "security-reviewer"
    role: "安全审查"

routing:
  - pattern: ".*style|lint.*"
    target: "code-style-checker"
  - pattern: ".*security|vuln.*"
    target: "security-reviewer"
  - pattern: ".*"
    target: "code-style-checker"
```

### 6.3 编排 SSE 事件

| event | 说明 |
|-------|------|
| `orchestration_start` | 开始 |
| `agent_start` | 子 Agent 启动 |
| `dispatch` | Supervisor 分发 |
| `agent_result` | 子 Agent 结果 |
| `aggregate` | 汇总 |
| `orchestration_done` | 结束 |

## Phase 7：Human-in-the-Loop（完整审批 UI）

**目标**：在 Phase 4 ToolGuard 之上，提供可配置的审批工作流与通知。

### 7.1 审批规则（Agent 级）

| 操作 | 默认 |
|------|------|
| write_file | 需审批 |
| execute_command | 需审批 |
| delete_file | 可选 |
| send_message | 通常不需要 |

配置：审批人、超时（如 5 分钟）、超时策略（拒绝/自动通过）、通知渠道（Web / 企微经 IntentGate）。

### 7.2 流程

```
Agent 遇敏感 tool → 暂停 → 创建 Approval 记录
    → 通知审批人 → 批准/拒绝/超时
    → 恢复或中止 Agent 执行
```

### 7.3 前端

待审批队列、上下文查看、批准/拒绝、历史记录。

## Phase 8：监控、会话检索与 Scheduler

**目标**：开发者调试 Agent，管理员监控系统；**定时 Agent 任务**支撑 IM 推送。

### 8.1 LangSmith

- 每 Agent 运行自动 Trace
- 自定义 Span（tool、model、orchestration）
- 错误 Trace 标记

### 8.2 Dashboard

- 今日对话数、活跃 Agent、Token 消耗、错误率
- Token 趋势（7 天）
- Agent 活跃排行
- 最近错误列表

### 8.3 会话检索

对标 Hermes SQLite FTS5，AetherMind 使用 PostgreSQL：

- 按 user_id、agent_id、时间、关键词搜索 Message
- 对话 replay API（调试/support）
- 导出 CSV/JSON（合规）

### 8.4 Scheduler

```yaml
# scheduler_jobs 示例
name: daily_sales_report
schedule: "0 9 * * 1-5"
agent_id: "uuid"
prompt: "生成本日销售摘要并推送卡片"
delivery:
  type: intentgate
  session_id: "wecom:default:..."
```

- APScheduler 或 Redis 延迟队列
- 执行：`AgentLoop.run_turn` → `card_intent` → IntentGate `aibot_send_msg`

### 8.5 日志

结构化 JSON 日志（loguru），级别：ERROR / WARNING / INFO / DEBUG。  
前端简易日志查看器（最近 1000 条）。

```
IM → IntentGate → AetherMindBackend → /api/agents/{id}/chat
```
