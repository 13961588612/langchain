### 启动方式

```bash
# 1. 进入 aethermind 目录
cd practice/aethermind

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，至少填入:
#   OPENAI_API_KEY=sk-xxx
#   (可选配置其他 Provider Key)

# 3. 启动所有服务
docker-compose up -d

# 4. 访问
#   前端:       http://localhost:3000
#   后端文档:   http://localhost:8000/docs
#   健康检查:   http://localhost:8000/api/health
```

### API 端点

#### 智能体 (Agents)

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/agents` | 创建智能体 |
| `GET` | `/api/agents` | 列表查询（`?page=&page_size=&search=&is_active=`） |
| `GET` | `/api/agents/{id}` | 获取详情 |
| `PUT` | `/api/agents/{id}` | 更新配置 |
| `DELETE` | `/api/agents/{id}` | 软删除 |
| `POST` | `/api/agents/{id}/activate?active=true` | 激活/停用 |

#### 对话 (Conversations)

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/conversations` | 创建对话 `{"agent_id","title"}` |
| `GET` | `/api/conversations` | 对话列表（`?agent_id=&page=&page_size=`） |
| `GET` | `/api/conversations/{id}` | 对话详情 |
| `DELETE` | `/api/conversations/{id}` | 删除对话及消息 |
| `GET` | `/api/conversations/{id}/messages` | 消息历史（`?limit=`） |
| `POST` | `/api/conversations/{id}/chat` | **SSE 流式对话** `{"message"}` |

#### 系统

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/health` | 健康检查 + Token 用量统计 |

#### 流式对话事件格式

SSE 流式响应包含以下事件类型：

| event | 说明 | content |
|-------|------|---------|
| `token` | LLM 输出的单个 token | 逐字内容 |
| `tool_call` | Agent 调用工具 | `"Calling tool: {tool_name}"` |
| `tool_result` | 工具返回结果 | `"Tool result: {tool_name}"` |
| `done` | 对话完成 | 空字符串 |
| `error` | 错误 | 错误消息 |

### 核心技术实现

- **AgentRuntime** (`backend/app/core/agent_runtime.py`)
  - 使用 DeepAgents 的 `create_deep_agent` 动态构建智能体
  - 集成 `FilesystemBackend` 作为工作目录
  - 集成 `AsyncPostgresSaver` 作为 LangGraph Checkpointer
  - 自动加载工作目录下的 Skills
  - 支持 soul_config + profile_config 注入 system_prompt

- **ModelHub** (`backend/app/core/model_hub.py`)
  - 通过 `init_chat_model` 统一接口支持 5 种 Provider
  - OpenAI / Anthropic / Google Gemini / Azure / Ollama
  - 模型实例缓存 + API Key 自动注入

- **SSE Streaming**
  - 后端使用 `sse-starlette` 的 `EventSourceResponse`
  - 前端通过 `fetch` + `ReadableStream` 实时读取 token 流
  - 支持 Markdown 渲染和工具调用事件展示

- **数据库**
  - PostgreSQL + pgvector（支持后续向量存储扩展）
  - 6 个 ORM 模型（Agent / Conversation / Message / Skill / ModelConfig / Channel）
  - 异步 SQLAlchemy + asyncpg 驱动
